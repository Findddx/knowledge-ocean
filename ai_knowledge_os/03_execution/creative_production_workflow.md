# AI 视频 / 音乐 Production Workflow

## 写在前面：你要建设的不是“会用模型”，而是一个小型创作工作室的操作系统

这份 workflow 的目标，不是教你怎么在一个下午试完十个模型，而是把“会出结果”升级成“会组织创作”。截至 2026-04-22，AI 创作最常见的问题并不是工具不够强，而是作者没有 production 思维：没有统一意图、没有参考系统、没有镜头和声音的组织方法、没有版本纪律、没有复盘机制。结果就是每次都像第一次开始，每次都能生成一些东西，但很难形成作品与方法。

真正成熟的 AI 创作流程，应该像一个小型 studio，而不是像一个 prompt 赌博机。你要做的，不是把所有不确定性交给模型；相反，你要学会在正确的位置留下不确定性，在其他位置建立约束。导演不是那个“把答案一次写给模型的人”，导演是那个知道哪些变化值得保留、哪些变化必须被消灭的人。

## 1. 生产观先于工具观

开始具体 workflow 之前，先立三条原则。

### 原则 A：创作是意图管理，不是结果碰运气

很多初学者做 AI 视频和音乐时，第一反应是不断换 prompt，希望突然撞到一个“对的结果”。这在短期内会带来惊喜，但不会带来能力。能力来自你是否能把意图显化为 brief、角色设定、镜头卡、节奏目标、声音情绪和版本标准。

### 原则 B：参考比形容词更重要，编辑比第一次生成更重要

官方产品路线已经把这件事讲得很明白。OpenAI 的图像和视频工作流强调 input image、editing、reference；Runway 的 References Guide 把 image reference 作为核心入口；很多创作者教程都在强调先确定视觉锚点，再做镜头变化。这个趋势说明，production 的中心不是“形容词堆叠”，而是“锚点管理”。

### 原则 C：声音不是后期补丁，而是结构的一部分

画面先行、声音最后随便补，是最常见也最伤作品的错误。音乐、音效、语音、节奏和静默共同决定了作品的呼吸方式。你在视频项目里越早考虑声音，作品越不容易只剩“好看的动态素材”。

## 2. 视频 workflow：从一句想法到一条可剪的片子

视频 production 应该被理解为六个阶段，每个阶段都要有明确输出物。没有输出物，流程就只是主观感觉；一旦有了输出物，agent、模板和评估才有接入点。

## 2.1 阶段一：创意简报，而不是立刻出片

任何片子开始前，先回答五个问题。

1. 这条片是给谁看的。
2. 它要让观众获得什么感受。
3. 它为什么必须以视频而不是文字存在。
4. 它最短可以多短，最长不能多长。
5. 它的一个核心记忆点是什么。

这一步看起来不像 AI 技术，但它实际上决定了你后面大部分返工。FSDL 对机器学习项目的一个核心提醒是：问题定义如果错了，后面再强的技术都会高成本地放大错误。创作项目完全一样。一个 brief 写不清，后面只会生成出越来越多看似华丽、实则方向飘移的素材。

建议把创意简报压缩成一页，至少包含：

- 片名或项目代号。
- 一句话主题。
- 目标观众。
- 时长上限。
- 情绪曲线。
- 视觉关键词。
- 声音关键词。
- 不允许出现的风格偏移。

## 2.2 阶段二：世界观与风格圣经

在这一步之前，不要急着做大量视频。先做图像层面的 visual bible。图像成本更低，反馈更快，适合用来锁定世界观。

这个阶段至少要产出四类锚点。

### 角色锚点

角色不是一张图，而是一组稳定特征：年龄感、脸型、发型、服饰逻辑、情绪气质、动作习惯、声音气质。没有角色锚点，视频里最常见的灾难就是“每一镜都是不同的人”。

### 场景锚点

场景不只是一张背景，而是空间关系、材质气质、时代感、光线逻辑与色彩温度。优秀的场景设定会让镜头在变化时仍然保留同一宇宙的呼吸。

### 镜头语气锚点

你要明确片子偏向什么样的摄影态度：静观、追随、压迫、抒情、纪实、MV 式、广告式，还是带有游戏 CG 的演出感。没有镜头语气，模型再会动也只是动态屏保。

### 声音气候锚点

这一步很多人会跳过，但其实越早定义越好。它包括配乐大致走向、环境音密度、是否有旁白、节奏偏慢还是偏切碎。声音设定会反过来影响镜头长度与剪辑方式。

## 2.3 阶段三：分镜不是画画，而是把片子的逻辑切开

视频之所以难，是因为一个镜头本身不难，镜头之间的关系才难。分镜的价值，不在于把画面画得多精美，而在于逼你回答：每个镜头的任务是什么，它怎么接前后镜头，它凭什么存在。

建议为每个镜头建立一张 shot card，字段至少包括：

- 镜头编号。
- 时长目标。
- 叙事功能。
- 主体动作。
- 摄影机动作。
- 参考图或 first-frame。
- 必须保留的视觉元素。
- 连接前后镜头的方式。
- 声音提示。

当你这么做之后，视频生成就不再是“让模型给我一段很酷的动态画面”，而是“让模型为一个已经存在的镜头职能服务”。这是 production 视角和 demo 视角的根本差异。

## 2.4 阶段四：镜头生成，先保结构，再保一致性，最后追惊艳感

真正进入生成环节后，最容易犯的错误是顺序颠倒。很多人先追求细节和冲击，结果后面发现结构和连续性根本立不住。更成熟的顺序应该是：

1. 先让镜头完成它的叙事任务。
2. 再确认角色、道具、场景和光线的一致性。
3. 最后才追求纹理、复杂运动和表层惊艳感。

OpenAI 的视频与图像工作流，Runway 的 References 和 Gen-4 Guide，都在告诉你：reference 是提高稳定性的第一工具，不是锦上添花。对你来说，这意味着每个镜头都应尽量挂靠到已有锚点，而不是从纯文本随机开始。

这里有一个很实际的判断：截至 2026-04-22，官方视频工作流仍然以短镜头和可编辑片段为主。也就是说，制作一条完整视频更像“生成一组可剪素材”，而不是“一次性输出终版”。如果不接受这个现实，就会不停对模型提出错误期待，然后把问题误判成“模型还不够强”。

## 2.5 阶段五：音画拼装，作品在这里才第一次真正出现

很多生成式创作者把剪辑理解为“把好看的片段拼起来”。这会使作品陷入典型的 AI 短片症：镜头都不差，但合在一起没有呼吸、没有推进、没有情绪层次。真正的剪辑不是拼接，而是重新组织注意力。

这一阶段至少要做四件事。

### 决定节奏

节奏不是快慢，而是信息密度、留白、切换时机与动静对比。你必须判断哪里该让观众停留，哪里该让能量释放，哪里该突然收紧。

### 决定声音与画面的主从关系

有些片子是画面带声音，有些片子是声音拖着画面走。两者完全不同。一个成熟作者会知道哪一个时刻应该先听见，再看见；哪一个时刻应该先看到，再被声音确认。

### 建立重复与变化

好的作品通常不会把所有镜头都做成同样强度。它会用重复建立记忆，用变化制造推进。这在音乐里叫动机发展，在视频里也同样成立。

### 删除

AI 时代最稀缺的能力之一就是删除。因为你很容易得到太多“还不错”的素材，最后反而不舍得砍。真正的作品质量，往往取决于你删掉了多少并不必要的漂亮片段。

## 2.6 阶段六：复盘，把一次项目变成下一次项目的基础设施

如果每做完一条片子就直接开始下一条，那你只是在积累疲劳，不是在积累能力。每个项目结束后，至少要回答以下问题。

- 最稳定的控制点是什么。
- 最不稳定的环节是什么。
- 最耗时的是生成、筛选、还是后期。
- 哪类 reference 最有效。
- 哪类 prompt 或镜头说明最容易失真。
- 哪些环节已经适合 agent 化。
- 哪些输出应沉淀为模板、风格卡、角色卡或 checklist。

这一步是把“创作经验”转化为“知识资产”的关键环节。没有复盘，再多项目也只是散点经验。

## 3. 音乐与声音 workflow：不要从“整曲幻想”开始，要从结构控制开始

音乐工作流最容易被误学成“一键出歌”。这种学法的问题不在于它没趣，而在于它几乎不会让你真正理解音乐如何作为作品的结构骨架存在。更成熟的学法，是从 motif、段落、情绪变化和音色角色开始。

## 3.1 第一步：先决定音乐在项目中的职责

音乐可能承担很多不同职责：做情绪底色、做节奏推进、做角色主题、做空间氛围、做转场、做记忆钩子。你必须先知道这次音乐的任务是什么，否则它就会沦为“看起来应该有的 BGM”。

## 3.2 第二步：先做 8-16 小节的主题实验，而不是整曲

从学习效率上看，短结构最有价值。一个 8-16 小节的主题足以让你练：

- 旋律的可记忆性。
- 节奏型是否服务画面。
- 和声与音色情绪是否匹配。
- 不同编曲版本如何影响同一主题。

AudioCraft / MusicGen 的官方材料强调 controllable music generation，这一点很值得你记住。音乐生成真正的门槛，不在“有没有音乐”，而在“你能否围绕条件和约束塑造音乐”。

## 3.3 第三步：把音乐和音效分开设计，再决定它们如何交会

电影感和游戏感往往不来自单纯的配乐，而来自配乐、环境音、拟音、静默和人声之间的关系。AI 创作里尤其如此。因为如果一开始就让所有声音一起乱出，你只会得到情绪模糊的堆叠。

建议至少分成四轨思考：

- 主题音乐。
- 环境氛围。
- 关键动作音效。
- 人声 / 旁白 / 角色语音。

OpenAI 的 Text-to-Speech 与 Voice Agents 文档很适合建立一个现实观：不同目标需要不同 pipeline。你要实时交互，就追求低延迟和自然往返；你要可编辑、可审校、可做内容创作，就更需要链式处理、文本中间态和可回放资产。这不是技术细节，而是工作流哲学。

## 3.4 第四步：让声音在结构上服务镜头，而不是一直“好听”

创作里一个反直觉的事实是：持续好听，未必持续有效。声音真正重要的是“什么时候出现、什么时候收掉、什么时候留下缺口”。许多初学者最难学会的不是做声音，而是让声音在正确的地方不出现。

所以你在音乐 workflow 里要始终问自己：

- 这段音乐是在引导、填充，还是遮盖问题。
- 它有没有让画面更清楚，而不是更拥挤。
- 它是否拥有一个可以被观众记住的核心动机。
- 它是否在关键时刻让位给了镜头、对白或静默。

## 4. Agent 在创作流程中的正确位置

创作型 agent 很容易被想象成“替你做作品的人”，但更现实、更高价值的位置通常是副导演、研究助理、制作统筹和档案管理员。

你完全可以让 agent 深度参与这些工作：

- 检索参考案例，按风格和镜头语言分类。
- 把创意简报扩展成镜头表与版本计划。
- 根据参考图总结角色与场景规则。
- 对生成结果做初步一致性核查。
- 维护 production log、版本变更和失败样例库。
- 为项目输出复盘报告、提示词演化记录和素材目录。

但有些事情仍然应由人主导。

- 最终审美取舍。
- 叙事与节奏判断。
- 哪些不稳定性值得保留为惊喜。
- 哪些“漂亮结果”其实破坏了作品核心。

简化地说，agent 最适合接管高频、重复、需要耐心的认知劳动；人最该保留的是方向、标准和删减权。

## 5. 一个可执行的 14 天最小创作冲刺

如果你现在就要开始，不必等“准备万全”。但要有节奏。

### 第 1-2 天：创意简报与参考收集

把主题、观众、风格、时长、声音关键词写清楚，建立参考板，不做大规模生成。

### 第 3-4 天：角色与场景基线

先在图像层锁定人物、色彩、材质、构图气质和世界观，再把可复用的元素保存为 bible。

### 第 5-6 天：分镜与镜头卡

把项目切成 6-12 个镜头，每个镜头有明确任务和参考，不要只写大段描述。

### 第 7-10 天：镜头生产与筛选

先出可用镜头，再做第二轮一致性修整，不要一开始沉迷于纹理级完美。

### 第 11-12 天：音乐、音效、语音

做主题动机、环境音、关键动作音，决定声音何时主导、何时退场。

### 第 13 天：剪辑

删掉冗余镜头，调整节奏，让片子第一次完整呼吸。

### 第 14 天：复盘

写下控制点、返工点、最有效的 reference、最无效的尝试，并更新模板。

## 6. 常见失败模式

### 失败模式 A：一开始就疯狂生成

这是最浪费时间的一种做法。没有 brief 和 shot list 的生成，只会把随机性放大。

### 失败模式 B：把所有问题都交给 prompt

很多问题本质上不是 prompt 问题，而是 reference、分镜、角色锚点、声音结构或剪辑判断的问题。

### 失败模式 C：忽视声音

没有声音结构，视频就算画面好看，也很难真正成立。

### 失败模式 D：舍不得删

AI 时代素材太多，删除就变成最稀缺的创作能力之一。

### 失败模式 E：没有项目归档

没有 production log，你永远不知道自己为什么这次成功、上次失败。

## 7. 模板层：把一次灵感变成可复用资产

到这里，workflow 才刚刚完成一半。真正决定你下一个项目会不会更顺手的，不是这次有没有碰巧出一个好镜头，而是你有没有把创作经验压缩成稳定模板。OpenAI 当前的视频 API 已经把 `input_reference`、`characters`、`edits`、`extensions` 分得很清楚；Runway 也把 reference 变成了可以保存、复用、共享的资产对象。这说明 production 的真实工作单位不是“灵感 prompt”，而是“约束、资产与评估”。

下面四张母表，足够支撑你把多数短视频 / 音乐项目做成可管理系统。

### 7.1 Creative Brief 模板

这张表的目标，不是写得漂亮，而是让项目在开始前就拥有边界。它最好控制在一页内，超过一页通常说明你还没有想清楚。

- `Project code`：项目代号。不要用情绪化名字，先用稳定命名，方便版本管理。
- `One-line thesis`：一句话主题。它不是剧情简介，而是这条作品真正想说的东西。
- `Why now`：为什么现在做。它回答的是项目动机，不是市场鸡汤。
- `Audience`：谁会看、在什么场景看。不同分发场景决定时长、节奏和信息密度。
- `Required emotion`：观众离开时应该留下什么情绪，不超过 3 个词。
- `Single memory beat`：观众只记住一个瞬间时，希望记住什么。
- `Runtime ceiling`：时长上限。不是目标时长，而是不可超过的天花板。
- `Format and delivery`：横屏 / 竖屏、是否带字幕、是否要旁白、最终发布介质。
- `Reference pack`：3-8 个参考，注明它们分别借什么，不允许笼统写“喜欢这种感觉”。
- `Sound mandate`：声音在本片中的职责，是推进、衬底、记忆点，还是空间感。
- `Continuity constraints`：哪些东西绝不能漂移，比如角色身份、光线逻辑、配色温度、镜头气质。
- `Hard bans`：明确禁止出现的俗套、审美漂移、镜头习惯或音效风格。
- `Definition of done`：什么叫完成。必须包括画面、声音、结构三条，不允许只写“好看”。

更重要的是字段背后的顺序。你应先定义 `One-line thesis`、`Single memory beat` 和 `Runtime ceiling`，再去写视觉和声音，因为风格是服务结构的，不是反过来。

### 7.2 Style Bible 模板

如果 brief 决定“拍什么”，style bible 决定“这个世界按什么规则存在”。它不是 moodboard 的别名，而是一个世界观约束文件。

- `World law`：这个世界的物理感、时代感、质感是否写实，还是更像梦境、游戏、广告、纪录片。
- `Character canon`：角色的稳定锚点。包括年龄感、轮廓、服装逻辑、表情张力、动作习惯、声音气质。
- `Environment canon`：空间结构、材质、光源逻辑、常见色温、道具密度。
- `Lens and framing grammar`：常用景别、视角高度、摄影机距离感，是否偏静观或追随。
- `Motion grammar`：镜头运动允许什么，不允许什么。是否允许高速漂移、突然变焦、过强的机械运动感。
- `Lighting and palette`：主色、辅色、反差等级、日夜逻辑、阴影是否锐利。
- `Texture rules`：颗粒、柔焦、噪点、锐度、皮肤或材质的处理方式。
- `Sound climate`：音乐、环境音、拟音、语音的总体密度与呼吸方式。
- `Approved variations`：允许变化的范围。比如同一角色可换服装，但不能换骨相；场景可换机位，但不能换光线逻辑。
- `Anti-drift list`：最容易失真的地方，以及发现失真后优先修什么。

这张表的作用，是让你在多个 shot 之间维持“同一宇宙”的感受。Sora 的 prompting guide 反复强调一致的措辞、稳定的光线逻辑和明确的 framing；Runway 的 reference workflow 则进一步把这种一致性变成资产管理问题。style bible 的使命，就是把这种一致性提前写死一部分。

### 7.3 Shot Card 模板

一张 shot card 不应只是“这镜头长什么样”，而应明确“这镜头为什么必须存在”。真正成熟的镜头卡会把结构、运动和评估同时写进去。

- `Shot ID`：如 `S03`。编号必须和时间线一致。
- `Narrative job`：这镜头是建立空间、推进动作、转情绪，还是制造记忆点。
- `In / out relation`：它怎样承接上一镜，又怎样把信息递给下一镜。
- `Duration target`：目标时长，不要先写“随便看看再说”。
- `Subject action in beats`：把动作拆成拍点。按照 Sora 当前指南，一镜最好只有一个清晰摄影机动作和一个清晰主体动作，再用拍点控制节奏。
- `Camera move`：推、拉、跟、摇、移、静止，只写一个主动作。
- `Frame and lens feel`：广角、近景、低机位、俯视等，用摄影语言保持稳定。
- `Reference assets`：挂靠的角色图、场景图、前一镜、输入图或角色资产。
- `Continuity carry-over`：必须继承的服装、道具位置、光向、配色、视线方向。
- `Edit note`：如果这镜头基于已有 clip 微调，明确只改一件事。OpenAI 的视频编辑接口目前最适合做单一、聚焦的修改。
- `Sound cue`：本镜需要听到什么，或必须保持什么静默。
- `Failure risk`：最可能崩的点是什么，例如肢体失真、角色漂移、镜头语气断裂。
- `Acceptance test`：什么情况下判定“可进剪辑”。例如主体身份稳定、动作可读、能与相邻镜头切上。

镜头卡的真正作用，是把“好不好看”拆成“能否剪、能否接、能否解释”。这会让你对素材的判断从审美直觉，提升到 production 判断。

### 7.4 Postmortem 模板

项目结束后的复盘不能写成情绪日记。更好的方式，是把复盘写成下一次生产的基础设施。

- `Initial thesis`：一开始你想验证什么。
- `What shipped`：最后真正交付了什么，不要把未完成部分也算进成果。
- `Strongest control point`：最有效的控制变量是什么，是 reference、角色资产、镜头卡，还是后期编辑。
- `Biggest failure mode`：返工最多的环节是哪一个，为什么。
- `Time breakdown`：时间到底花在检索、筛选、生成、编辑、声音还是剪辑。
- `Best reference`：哪类参考最有效，为什么。
- `Worst instruction`：哪类 prompt 或镜头说明最容易导致漂移。
- `Human judgment moments`：哪些关键判断必须由人做，不能委托给 agent。
- `Reusable assets created`：哪些东西值得沉淀为模板、角色卡、风格卡、命名规范或评估表。
- `Kill / keep / try next`：下次立刻删除什么、保留什么、只值得小规模再试什么。

如果你愿意再严格一步，可以给复盘加一个 10 分制总评分，但要配套“示例锚点”。OpenAI 的 evaluation best practices 明确建议评分要配样例、阈值和多轮校准，否则数字会快速退化成假精确。

## 8. 三组 drills：把方法论练成肌肉，而不是停留在概念上

模板解决“知道该做什么”，drill 解决“能不能稳定做到”。下面三组练习都故意做得很窄，因为 production 能力不是靠宏大项目练出来的，而是靠受控练习积累起来的。

### 8.1 Reference Lock Drill

目标：训练“先锁世界，再出镜头”的习惯。

- 时长：2 天。
- 约束：只允许做图像，不允许先做视频。
- 输入：3 个角色参考、3 个环境参考、1 个 brief。
- 输出：一页 style bible、1 套角色卡、4 张同一世界观下的场景图。
- 通过标准：陌生人只看图片，也能判断这些图属于同一项目。
- 失败信号：每张图都好看，但像来自不同片子；或者角色身份在 4 张图里已经换了人。

这个练习的价值，是把“参考”从灵感收藏夹升级成控制系统。

### 8.2 One-Clear-Move Drill

目标：训练镜头动作与主体动作的清晰度。

- 时长：1 天。
- 约束：每个 shot 只允许一个摄影机动作、一个主体动作、一个情绪目标。
- 输入：同一角色资产和同一场景锚点。
- 输出：4 个 3-6 秒短镜头，分别承担建立、推进、转折、收束四种叙事职能。
- 通过标准：去掉文字说明，旁观者仍能说出每镜大概在做什么。
- 失败信号：镜头看起来很热闹，但无法概括；或者一镜里塞了太多动作导致没有主句。

它直接训练的是 shot card 思维。很多所谓“模型不稳定”，其实是镜头任务不清。

### 8.3 Sound-First Drill

目标：防止把声音降格为最后的装饰。

- 时长：2 天。
- 约束：先做 20-30 秒声音草图，再据此决定镜头切法。
- 输入：一个情绪 brief 和一个主题动机。
- 输出：1 条声音时间线、3-5 个镜头切点说明、一个短成片。
- 通过标准：把画面关掉，只听声音也能感到起承转合。
- 失败信号：声音只能填空，无法决定结构；或者画面和声音像两条互不相认的轨。

这个练习尤其重要，因为大多数 AI 创作者天然偏画面，而真正的“作品感”常常是在声音层第一次成立。

## 9. 两个短案例：好项目和坏项目不是审美高下，而是组织能力高下

### 案例 A：一个好项目长什么样

项目名：`灯塔管理员的夜巡`。

- 目标：做一条 35 秒竖屏预告，核心记忆点是“风暴中提灯转身”的瞬间。
- 边界：一个角色、一个场景、6 个镜头、一个主题动机、10 天内结束。
- 控制策略：先用图像锁角色和灯塔内部空间，再用 6 张 shot card 生产素材，声音只保留风声、脚步和低频动机。
- 为什么好：它把 ambition 压在“气质完成度”和“连续性”上，而不是压在角色数量和世界观规模上。它练的是 production intelligence，不是堆料。

### 案例 B：一个坏项目长什么样

项目名：`做一个四分钟的史诗级 AI 短片`。

- 目标：同时展示科幻世界观、两条人物线、完整剧情、配音、原创音乐、复杂动作戏。
- 边界：没有明确边界，只写“先试试看”。
- 控制策略：几乎没有，主要靠不停换 prompt。
- 为什么坏：问题不在野心大，而在于它没有把任何关键约束前置。没有 reference system、没有时长封顶、没有镜头职责、没有声音设计，也没有“什么叫完成”。这种项目最常见的结局不是失败，而是永远处于“好像快成了”的假进展。

好项目和坏项目的差别，常常不是模型版本，而是你有没有把不确定性切碎并分配到合适的载体里。

## 10. 结论：创作 workflow 的本质，是把灵感变成可管理的不确定性

AI 创作并没有取消导演、制作人、音乐总监、剪辑师这些角色的逻辑；它只是把这些角色的边界重新压缩到了一个人或一个小团队身上。因此，真正值得学习的从来不是“某个模型会不会替代创作者”，而是“创作者能否借助模型拥有过去需要一个团队才能拥有的生产杠杆”。

你应该把每个项目当成一次能力组织实验。作品当然重要，但更重要的是：你有没有因此建立更清楚的审美标准、更好的流程模板、更可复用的资产库，以及更可靠的自我判断。只有这样，AI 创作才不是短暂兴奋，而是长期生产力。
